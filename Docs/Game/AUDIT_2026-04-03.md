# NovaForge ‚Äî Project Audit 2026-04-03

## WPF Status
There is no WPF code in this repository. The repo is 100% C++/CMake.
Historical chat rollup ZIPs (which documented the earlier C++ patch integration effort)
have been moved to `Archive/`. Nothing WPF-related was ever committed to source.

## Repo Boundary
- **NovaForge** owns: game engine, voxel runtime, game-specific editor, NovaForge game content.
- **Does not own**: Atlas Suite, generic workspace tooling, any .NET/WPF layer.
- Build: `cmake -B build -DCMAKE_CXX_STANDARD=20 && cmake --build build`
- Outputs: `build/bin/NovaForgeEditor.exe`, `build/bin/NovaForgeGame.exe`

## What Is Incomplete (Next Critical Path)

### Priority 1 ‚Äî Voxel edit persistence
- **`DevWorldSerializer` round-trip not smoke-tested.** Save/Load commands exist but a save-then-reload cycle has not been verified to preserve voxel edits.
- **Task**: run one edit ‚Üí `World.SaveDevWorld` ‚Üí `World.ReloadDevWorld` ‚Üí verify chunk matches.

### Priority 2 ‚Äî Voxel undo/redo
- **`CommandHistory` exists but voxel edits are not pushed to it.** The `Edit.Undo`/`Edit.Redo` commands are registered and `CommandHistory` is functional, but `VoxelAdd` and `VoxelRemove` tool actions bypass it.
- **Task**: wrap `VoxelEditApi.SetVoxel` and `VoxelEditApi.Mine` calls in reversible `IEditorCommand` objects and push them to `m_CommandHistory`.

### Priority 3 ‚Äî Inspector property editing
- **Inspector is read-only.** It correctly displays selected voxel position and type, and entity IDs, but there are no editable fields ‚Äî no way to change a value and have it push back into the world.
- **Task**: add simple editable fields (at minimum: voxel type selector when a voxel is selected) that call through `VoxelEditApi`.

### Priority 4 ‚Äî Scene Outliner shows "(empty world)"
- **The ECS `World` has no entities.** The outliner is correctly wired to `Level.GetWorld().GetLiveEntities()` but that list is always empty because no entities are spawned into the ECS. The visible voxel terrain lives in `ChunkMap`, not ECS.
- **Task**: either spawn representative ECS entities for chunks/spawn-points in `GameWorld::Initialize`, or add a separate chunk-list section to the outliner.

### Priority 5 ‚Äî Remove inactive viewport path
- **`Source/Editor/Viewport/EditorViewportNF.cpp` (nf:: namespace)** is a leftover parallel viewport implementation that is not used by `EditorApp`. It should be archived.
- **Task**: move `EditorViewportNF.cpp/.h` to `Archive/` or an `Archived/` source folder, and remove from `CMakeLists.txt`.

### Priority 6 ‚Äî Toolbar enable state not derived from command availability
- Commands have a `CanExecute` predicate but the toolbar buttons do not call it. Undo/Redo buttons are always clickable even when there is nothing to undo.
- **Task**: have `DrawButton` accept an optional `enabled` flag, call `CanExecute` for Undo/Redo, and dim the button when disabled.

### Priority 7 ‚Äî NovaForgeGame.exe not built alongside editor in VS workflow
- `Launch Game` now logs the correct error code. Both targets output to `build/bin/` under CMake but VS `Debug/` builds put them separately. Users need to build `NovaForgeGame` before `Launch Game` will work.
- **Task**: document in README; optionally add a post-build copy step.
