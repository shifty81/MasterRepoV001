# NovaForge — Project Audit 2026-04-03

## WPF Status
There is no WPF code in this repository. The repo is 100% C++/CMake.
Historical chat rollup ZIPs (which documented the earlier C++ patch integration effort)
have been moved to `Archive/`. Nothing WPF-related was ever committed to source.

---

## Repo Boundary
- **NovaForge** owns: game engine, voxel runtime, game-specific editor, NovaForge game content.
- **Does not own**: Atlas Suite, generic workspace tooling, any .NET/WPF layer.
- Build: `cmake -B build -DCMAKE_CXX_STANDARD=20 && cmake --build build`
- Outputs: `build/bin/NovaForgeEditor.exe`, `build/bin/NovaForgeGame.exe`

## What Is Incomplete (Next Critical Path)

### Priority 1 — Voxel edit persistence
- **Implemented.** Save command now writes both entity header (`DevWorld.nfsv`) and chunk voxel data (`DevWorld.nfck`). Reload command loads saved chunk data after re-initializing. Round-trip test added (326 tests passing). `GameWorld::Shutdown()` now clears the ChunkMap. World dirty flag resets after successful save.

### Priority 2 — Voxel undo/redo
- **Implemented.** `VoxelPlaceCommand`, `VoxelRemoveCommand`, and `VoxelTypeEditCommand` all push through `CommandHistory`. `Edit.Undo`/`Edit.Redo` commands registered with CanExecute predicates.

### Priority 3 — Inspector property editing
- **Implemented.** Voxel "Type" property is editable in the property grid via click (increment/decrement). Changes write back to the world through `VoxelTypeEditCommand` and participate in undo/redo. Dirty state propagates correctly.

### Priority 4 — Scene Outliner shows chunk data
- **Implemented.** `RebuildWorldOutliner()` feeds live chunk metadata (coord, dirty flags) into `SceneOutliner::SetChunkData()` every frame. The outliner shows the world tree with chunk entries.

### Priority 5 — Inactive viewport archived
- **Done.** `EditorViewportNF.cpp/.h` moved to `Archive/Examples/Editor_nf_layer/` and removed from CMakeLists.

### Priority 6 — Toolbar enable state derived from command availability
- **Done.** Undo/Redo toolbar buttons call `CanExecute` and dim when disabled. Menu drop-down items also honor command availability.

### Priority 7 — NovaForgeGame.exe not built alongside editor in VS workflow
- `Launch Game` now logs the correct error code. Both targets output to `build/bin/` under CMake but VS `Debug/` builds put them separately. Users need to build `NovaForgeGame` before `Launch Game` will work.
- **Task**: document in README; optionally add a post-build copy step.
