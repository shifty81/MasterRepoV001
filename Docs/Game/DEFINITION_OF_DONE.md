# MasterRepo Reset — Definition of Done

## Reset Complete When
- docs all point to the same phase order
- repo boundary is unambiguous
- editor and game both build from clean checkout
- the project manifest/config exists in repo
- dev world is the only active world target for early milestones
- voxel runtime is explicitly phase-gated after dev world bootstrapping
- first interaction loop is explicitly phase-gated after voxel runtime

## Not Done When
- roadmap still mixes old and new milestone order
- docs still imply Suite or Atlas ownership
- phase goals overlap without boundaries
- build output still lacks a working editor or game executable
