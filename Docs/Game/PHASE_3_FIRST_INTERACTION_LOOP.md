# Phase 3 — First Interaction Loop Only

## Goal
Prove a small but real gameplay loop.

## Loop Shape
1. spawn into dev world
2. use starter R.I.G.
3. gather or mine a resource
4. place, repair, or build a simple object
5. receive minimal HUD/status feedback
6. restart and confirm persistence or repeatability

## Required Systems
- starter player state
- starter R.I.G. data/state
- one interaction tool
- one resource type minimum
- inventory baseline
- simple placement or repair action
- minimal HUD widgets
- result logging for smoke tests

## Definition of Done
- the loop is playable in editor test mode
- the loop is playable in the standalone game client
- the player can repeatedly complete the loop without a hard crash
- the R.I.G. starter state is visible in data and gameplay behavior
